# Итоговый отчёт тестирования сервиса проверки стойкости паролей

**Проект:** Сервис проверки стойкости паролей и их присутствия в базах утечек  
**Версия:** MVP  

---

## 1. Введение

Сервис предназначен для проверки паролей на стойкость и поиска их в базе утечек.  
Задачи тестирования:
- проверить корректность оценки стойкости,
- проверить работу с базой утечек,
- проверить обработку ошибок,
- проверить безопасность,
- проверить удобство интерфейса.

---

## 2. Функциональные проверки

Цель: убедиться, что сервис правильно оценивает пароли и находит их в базе утечек.

### 2.1. Проверка уровня стойкости

| Пароль | Ожидаемый уровень | Фактический результат |
|--------|-------------------|------------------------|
| `123456` | Слабый | Слабый |
| `Password1` | Средний | Средний |
| `MyStr0ngP@ssw0rd!2024` | Сильный | Сильный |

### 2.2. Проверка утечек

| Пароль | Ожидание | Фактический результат |
|--------|----------|------------------------|
| `qwerty123` | "Обнаружен в утечках" | Обнаружен |
| `admin123` | "Обнаружен в утечках" | Обнаружен |
| `123456789` | "Обнаружен в утечках" | Обнаружен |
| `MyStr0ngP@ssw0rd!2024` | "Не обнаружен" | Не обнаружен |

**Вывод:** функциональность работает в соответствии с ожиданиями.

---

## 3. Проверка ограничений ввода

Цель: проверить, как сервис реагирует на некорректные пароли.

| Проверка | Входные данные | Результат |
|----------|----------------|-----------|
| Пустой пароль | (ничего) | Кнопка неактивна |
| Один пробел | `" "` | Ошибка: "Пароль не может состоять из пробелов" |
| Три пробела | `"   "` | Ошибка: "Пароль не может состоять из пробелов" |
| 128 символов | `a×128` | Обработан корректно |
| 129 символов | `a×129` | Рекомендация уменьшить длину |

**Вывод:** ограничения работают. Для паролей длиннее 128 символов рекомендуется сразу выдавать ошибку без анализа.

---

## 4. Проверка безопасности

Цель: убедиться, что сервис защищён от атак и не хранит пароли в открытом виде.

| Проверка | Результат |
|----------|-----------|
| SQL-инъекция (`admin' OR '1'='1`) | Не прошла |
| XSS-атака (`<script>alert(1)</script>`) | Не прошла |
| Хардкод паролей в коде | Не найдено |
| Хранение паролей в БД | Только хэши |
| HTTPS | Используется HTTP |

**Вывод:** основные угрозы отсутствуют. Рекомендуется добавить HTTPS перед публичным запуском.

---

## 5. Проверка UI/UX

Цель: проверить удобство интерфейса и поведение на разных устройствах.

| Проверка | Результат |
