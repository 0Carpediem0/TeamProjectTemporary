# Развёртывание приложения

## 1. Перенос проекта на сервер

На сервер необходимо перенести рабочую копию проекта. Это можно сделать разными способами:

- через `git clone`
- через `scp`
- через `rsync`
- через загрузку архива и распаковку

Для запуска приложения необходимы как минимум:

- `backend/`
- `frontend/`
- `requirements.txt`
- `README.md`
- локальная база утечек из `backend/data/`

## 2. Создание окружения и установка зависимостей

После переноса проекта нужно перейти в корень приложения и установить зависимости.

Пример:

```bash
cd /path/to/project
python3 -m virtualenv .venv
. .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

## 3. Запуск приложения

Текущий проект запускается через `uvicorn`.

Пример команды:

```bash
cd /path/to/project
. .venv/bin/activate
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

Для фонового запуска можно использовать `nohup`:

```bash
cd /path/to/project
nohup ./.venv/bin/python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 > app.log 2>&1 &
```

## 4. Проверка работы

После запуска рекомендуется проверить:

- открывается ли главная страница
- отвечает ли `POST /api/check`
- загружаются ли статические файлы

Примеры:

```bash
curl -I http://127.0.0.1:8000/
```

```bash
curl -X POST http://127.0.0.1:8000/api/check \
  -H "Content-Type: application/json" \
  -d '{"password":"12345678"}'
```

## 5. Перезапуск после изменений

Если в проект были внесены изменения, удобный общий порядок такой:

1. остановить старый процесс
2. обновить файлы проекта
3. при необходимости обновить зависимости
4. запустить приложение снова

Пример остановки:

```bash
pkill -f "uvicorn backend.main:app"
```

Пример просмотра логов:

```bash
tail -f app.log
```
