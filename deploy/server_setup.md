# Подготовка сервера

## Минимальные требования

Для развёртывания проекта требуется сервер со следующими характеристиками:

- Linux-система
- доступ по SSH
- Python 3.10 или новее
- доступ в интернет для установки зависимостей
- открытый порт для веб-приложения

## Базовая подготовка

Перед загрузкой проекта рекомендуется:

1. Обновить список пакетов и установить инструменты Python.
2. Подготовить рабочую директорию для проекта.
3. Убедиться, что на сервере можно запускать `python3`, `pip` и создавать виртуальное окружение.

Пример команд:

```bash
sudo apt-get update
sudo apt-get install -y python3 python3-pip
python3 -m pip --version
```

Если стандартный модуль `venv` недоступен, можно использовать альтернативный вариант через `virtualenv`.

Пример:

```bash
python3 -m pip install -U virtualenv
python3 -m virtualenv .venv
```

## Рекомендуемая структура на сервере

Проект удобно хранить в отдельной директории, например:

```text
/opt/project-name
```

или

```text
/home/<user>/project-name
```

Главное, чтобы у пользователя запуска были права на:

- чтение исходного кода
- запись логов
- создание виртуального окружения

## Что проверить до деплоя

- сервер доступен по SSH
- Python запускается без ошибок
- `pip` установлен
- директория для проекта создана
- порт приложения не занят другим процессом
