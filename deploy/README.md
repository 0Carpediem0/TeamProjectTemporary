# Deploy

В этой папке собраны универсальные материалы по развёртыванию проекта на удалённом Linux-сервере без привязки к конкретному хостингу или учебной инфраструктуре.

## Что находится в папке

- `server_setup.md` — базовая подготовка сервера
- `app_deploy.md` — загрузка проекта и запуск приложения
- `production_notes.md` — практические замечания по текущему проекту
- `deploy_checklist.md` — короткий чек-лист перед выкладкой

## Общая логика развёртывания

1. Подготовить сервер и установить Python-инструменты.
2. Перенести проект на сервер.
3. Создать изолированное окружение и установить зависимости.
4. Запустить FastAPI-приложение.
5. Проверить доступность веб-интерфейса и API.
6. При необходимости оформить автозапуск и перезапуск сервиса.

## Для какого сценария подходят инструкции

Документация рассчитана на учебный или демонстрационный деплой проекта, в котором:

- backend запускается на `FastAPI`
- frontend отдаётся тем же приложением
- база утечек хранится локально в виде SQLite-файла
- приложение разворачивается на собственном VPS или учебном Linux-сервере

## Что важно

Документация ниже описывает общий подход. Конкретные адреса серверов, учётные данные, SSH-ключи и внутренние сетевые параметры в репозиторий не добавляются.
