# Практические замечания

## Назначение текущего решения

Текущий способ развёртывания подходит прежде всего для:

- учебной демонстрации
- тестового запуска
- внутреннего показа функциональности

Для полноценного production-сценария проекту потребуется дополнительная подготовка.

## Что важно учесть для этого проекта

### 1. Конфиденциальные данные

Учётные данные сервера, SSH-ключи, пароли, внутренние адреса и токены не должны храниться в репозитории.

### 2. База утечек

В проекте используется локальная SQLite-база с хэшами паролей. Важно сохранять этот принцип и не хранить исходные пароли в открытом виде.

### 3. Логи приложения

Перед постоянным запуском рекомендуется проверить, что:

- приложение не пишет пароль в открытом виде
- уровень логирования не избыточен
- debug-настройки отключены

### 4. Конфигурация запуска

Для более устойчивого сценария желательно:

- запускать приложение от отдельного пользователя
- оформить сервис через `systemd`
- добавить автоматический перезапуск при падении

### 5. Конфигурация приложения

Перед публичным или длительным запуском стоит вынести переменные окружения для:

- строки подключения к БД
- режима логирования
- порта приложения

### 6. Проверка безопасности

Перед деплоем рекомендуется дополнительно убедиться, что:

- чувствительные данные не попадают в API-ответы
- база данных не открыта в общий доступ
- серверный порт открыт только там, где это действительно нужно
