# TeamProjectTemporary
Временный репозиторий для работы над проектом "Разработка сервиса проверки стойкости паролей и их присутствия в базах утечек"

## Текущее состояние проекта
На текущем этапе проект находится в рабочем состоянии MVP. Уже реализованы:

- backend на `Python + FastAPI`;
- endpoint `POST /api/check` для проверки пароля;
- базовая логика оценки стойкости пароля;
- frontend-интерфейс для ввода пароля и отображения результата;
- проверка пароля по базе хэшей утечек;
- базовые тестовые и отчётные материалы.

Сервис можно запускать локально и использовать для демонстрации основного пользовательского сценария:

1. пользователь вводит пароль;
2. система анализирует его по набору правил;
3. дополнительно проверяется наличие пароля в базе утечек;
4. пользователь получает итоговую оценку и пояснение причин.

## Общее описание
Проект представляет собой веб‑приложение для проверки стойкости пароля.  
Оценка выполняется по следующим критериям:
- длина пароля;
- состав (цифры, буквы в разных регистрах, спецсимволы);
- отсутствие запрещённых последовательностей (простые/клавиатурные/популярные пароли, повторы, даты);
- отсутствие пароля в базе утекших хешей.

Результат проверки отображается пользователю в виде **«Слабый» / «Средний» / «Сильный»** с пояснением причин.

## MVP (Minimum Viable Product)

Минимально жизнеспособный продукт включает:

1. **Веб-интерфейс** — одна страница с полем ввода пароля и кнопкой проверки
2. **API проверки стойкости** — анализ пароля по критериям:
   - длина (минимум 8 символов)
   - наличие цифр, строчных и заглавных букв, спецсимволов
   - отсутствие простых/клавиатурных/популярных последовательностей
   - отсутствие повторов и дат
3. **База утекших хешей** — проверка пароля по SQLite-базе (только хеши, не пароли)
4. **Вывод результата** — оценка «Слабый» / «Средний» / «Сильный» с пояснением причин

## Быстрый запуск

### 1. Проверка окружения

Убедитесь, что установлен Python:

```powershell
python --version
```

Рекомендуется использовать Python 3.10+.

### 2. Установка зависимостей

Из корня проекта выполнить:

```powershell
pip install -r requirements.txt
```

### 3. Запуск приложения

Запустить backend-сервер:

```powershell
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000
```

### 4. Что проверить после запуска

После открытия страницы рекомендуется убедиться, что:

- загружается главная страница сервиса;
- отображаются стили и интерфейс ввода пароля;
- работает кнопка проверки;
- для слабых или скомпрометированных паролей отображается корректный результат;
- endpoint `/api/check` отвечает без ошибок.

## Использование

### Через браузер

1. Откройте `http://127.0.0.1:8000`
2. Введите пароль в поле ввода
3. Нажмите «Проверить пароль»
4. Изучите результат проверки в интерфейсе

### Через Swagger UI

FastAPI автоматически предоставляет документацию по адресу:

```text
http://127.0.0.1:8000/docs
```

Там можно вручную протестировать `POST /api/check`.

### Через PowerShell

```powershell
$body = @{password = "Test123!"} | ConvertTo-Json
Invoke-RestMethod -Uri "http://127.0.0.1:8000/api/check" -Method POST -Body $body -ContentType "application/json"
```

Пример ожидаемого ответа:

```json
{
  "strength": "medium",
  "scores": 3,
  "reasons": [
    "Избегайте последовательностей цифр (123, 1234)"
  ]
}
```
