# TeamProjectTemporary

Репозиторий проекта **«Разработка сервиса проверки стойкости паролей и их присутствия в базах утечек»**.

## Текущее состояние проекта

На текущем этапе проект находится в рабочем состоянии MVP. Уже реализованы:

- backend на `Python + FastAPI`;
- endpoint `POST /api/check` для проверки пароля;
- базовая логика оценки стойкости пароля;
- frontend-интерфейс для ввода пароля и отображения результата;
- проверка пароля по базе хэшей утечек;
- тестовые и отчётные материалы по итерациям.

Сервис можно запускать локально и использовать для демонстрации основного пользовательского сценария:

1. пользователь вводит пароль;
2. система анализирует его по набору правил;
3. дополнительно проверяет наличие пароля в базе утечек;
4. пользователь получает итоговую оценку и пояснение причин.

## Общее описание проекта

Проект представляет собой веб-приложение для проверки стойкости пароля.

Оценка выполняется по следующим критериям:

- длина пароля;
- состав пароля: цифры, буквы в разных регистрах, специальные символы;
- отсутствие небезопасных последовательностей и шаблонов;
- отсутствие пароля в базе утекших хэшей.

Результат проверки отображается пользователю в виде оценки:

- `weak`
- `medium`
- `strong`

и сопровождается пояснением причин.

## MVP

Минимально жизнеспособный продукт включает:

1. **Веб-интерфейс**
   Страница с полем ввода пароля и кнопкой запуска проверки.

2. **API проверки**
   Сервис принимает пароль и возвращает результат анализа.

3. **Проверка стойкости**
   Анализируются длина пароля, состав, шаблоны и небезопасные последовательности.

4. **Проверка по базе утечек**
   Пароль сравнивается с локальной базой хэшей утёкших паролей.

5. **Вывод результата**
   Пользователь получает уровень стойкости и список причин.

## Что уже реализовано

- проверка длины и состава пароля;
- выявление небезопасных последовательностей и шаблонов;
- проверка по базе SHA-256 хэшей утёкших паролей;
- вывод уровня стойкости: `weak / medium / strong`;
- отображение причин результата пользователю;
- пользовательский интерфейс и интеграция с API;
- базовые инструкции по локальному запуску и деплою.

## Локальный запуск

### 1. Проверка окружения

Убедитесь, что установлен Python:

```powershell
python --version
```

Рекомендуется использовать Python 3.10+.

### 2. Установка зависимостей

Из корня проекта выполните:

```powershell
pip install -r requirements.txt
```

### 3. Запуск приложения

```powershell
python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000
```

### 4. Что проверить после запуска

После открытия страницы рекомендуется убедиться, что:

- загружается главная страница сервиса;
- отображаются стили и поле ввода пароля;
- работает кнопка проверки;
- результат проверки отображается без ошибок;
- endpoint `/api/check` отвечает корректно.

## Использование

### Через браузер

1. Откройте `http://127.0.0.1:8000`
2. Введите пароль в поле ввода
3. Нажмите кнопку проверки
4. Изучите результат в интерфейсе

### Через Swagger UI

FastAPI автоматически предоставляет документацию по адресу:

```text
http://127.0.0.1:8000/docs
```

Там можно вручную протестировать `POST /api/check`.

### Через PowerShell

```powershell
$body = @{password = "Test123!"} | ConvertTo-Json
Invoke-RestMethod -Uri "http://127.0.0.1:8000/api/check" -Method POST -Body $body -ContentType "application/json"
```

Пример ожидаемого ответа:

```json
{
  "strength": "medium",
  "scores": 3,
  "reasons": [
    "Избегайте последовательностей цифр (123, 1234)"
  ]
}
```
