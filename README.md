# Тестирование сервиса проверки стойкости паролей

## Общее описание

В данном репозитории представлены материалы по тестированию сервиса проверки стойкости паролей.  
Тестирование включает проверку:
- длины пароля;
- состава (цифры, буквы, спецсимволы);
- наличия запрещённых последовательностей;
- проверки по базе утечек.

---

## Структура 

| Файл | Назначение |
|------|-----------|
| `правила_проверки.md` | Правила оценки стойкости пароля (баллы, штрафы, итоговая оценка) |
| `ручные_тесты.md` | Тест-кейсы для ручного тестирования с таблицей для заполнения результатов |
| `автотесты.py` | Данные для автоматизированного тестирования (пароль → ожидаемая оценка) |
| `безопасность.md` | Чек-лист безопасности (проверка логов, хранения, API, UI) |

---

## Как проводить тестирование

### Ручное тестирование

1. Открыть файл `ручные_тесты.md`
2. Запустить сервис проверки паролей
3. Вводить пароли из таблицы
4. Заполнять колонки «Фактический результат» и «Баг?»
5. Записывать найденные ошибки в раздел «Список найденных багов»

### Автоматизированное тестирование

1. Передать файл `автотесты.py` разработчику
2. Разработчик добавит вызов API и запустит автотесты
3. Все тесты должны проходить успешно

---

## Чек-лист безопасности

Файл `безопасность.md` содержит список проверок:
- пароль не сохраняется в БД
- пароль не пишется в логи
- API не возвращает пароль
- поле ввода скрывает символы
- передача только методом POST

Проверки выполняются по мере разработки и финально — перед сдачей MVP.
