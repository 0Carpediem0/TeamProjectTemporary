# ОТЧЁТ ПО ТЕСТИРОВАНИЮ (спринт 2)

## Что тестировалось
- Бэкенд API (`/api/check`)
- Безопасность (логи, БД, POST/GET)
- Фронтенд (поле ввода, консоль, localStorage)

## Как тестировалось
- Ручное тестирование через Postman
- Просмотр логов сервера
- Проверка фронта через F12 (Console, Application, Elements)

## Результаты

| Проверка | Результат |
|----------|-----------|
| Оценка пароля (1234567 → weak) | ✅ |
| Оценка пароля (Pass1234! → medium) | ✅ |
| Оценка пароля (StrongP@ss2025 → strong) | ✅ |
| Пустой пароль → weak (должна быть ошибка) | ❌ БАГ |
| GET-запрос → 405 Method Not Allowed | ✅ |
| Проверка утечек (leaked_password_123) | ✅ |
| Поле ввода скрывает символы (звёздочки) | ✅ |
| Пароль не в консоли браузера | ✅ |
| Пароль не в localStorage | ✅ |
| Пароль не в логах сервера | ✅ |
| В БД только хэши | ✅ |

## Найденные баги
1. **Пустой пароль** — сервер возвращает `weak`, а должен возвращать ошибку `400 Bad Request`

Бэкенд работает корректно, требования безопасности соблюдены. Фронт готов. Сервис готов к спринту 3.
