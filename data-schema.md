# Схема хранения данных для сервиса проверки паролей

## Общий подход

Сервис не хранит введённые пользователями пароли ни в каком виде. База данных содержит только **предварительно
вычисленные SHA‑256 хэши паролей, которые уже были скомпрометированы** (синтетический набор утечек). Это позволяет
быстро проверить, встречался ли пароль в утечках, без сохранения самого пароля.

## Выбор хэш-функции

Для проверки наличия пароля в базе утечек используется **SHA‑256**. Эта функция обладает следующими свойствами:

- **Детерминизм**: один и тот же пароль всегда даёт одинаковый хэш.
- **Отсутствие соли**: соль не применяется, так как цель — точное совпадение с заранее вычисленными хэшами утекших
  паролей. Соль использовалась бы для хранения паролей пользователей, но здесь хранятся только хэши известных утечек.
- **Фиксированная длина**: выход SHA‑256 составляет 256 бит, что в hex-представлении даёт ровно 64 символа.

## Структура таблицы

Единственная таблица `breached_hashes` хранит множество уникальных хэшей. Поле `id` служит первичным ключом, поле
`hash_password` содержит шестнадцатеричное представление хэша.

```sql
CREATE TABLE breached_hashes (
    id INTEGER PRIMARY KEY,
    hash_password VARCHAR(64) NOT NULL UNIQUE,
);
