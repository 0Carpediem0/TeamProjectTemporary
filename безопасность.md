# ЧЕК-ЛИСТ БЕЗОПАСНОСТИ

## Общие требования

Проект должен соблюдать следующие правила безопасности при обработке паролей.

---

## 1. Хранение данных

| № | Проверка | Статус | Примечание |
|---|----------|--------|------------|
| 1.1 | Пароль не сохраняется в базе данных | ☐ | |
| 1.2 | В базе утечек хранятся только хэши паролей, а не сами пароли | ☐ | |
| 1.3 | Нет файлов с логами, содержащими пароли | ☐ | |

---

## 2. Логирование

| № | Проверка | Статус | Примечание |
|---|----------|--------|------------|
| 2.1 | Пароль не выводится в консоль (print, console.log) | ☐ | |
| 2.2 | Пароль не записывается в файлы логов | ☐ | |
| 2.3 | В логах сервера нет текста введённого пароля | ☐ | |
| 2.4 | В логах браузера (F12) нет пароля | ☐ | |

---

## 3. API безопасность

| № | Проверка | Статус | Примечание |
|---|----------|--------|------------|
| 3.1 | API не возвращает пароль в ответе | ☐ | |
| 3.2 | Передача пароля только методом POST (не GET) | ☐ | |
| 3.3 | Пароль не передаётся в URL | ☐ | |
| 3.4 | При ошибке валидации не возвращается пароль | ☐ | |
| 3.5 | API использует HTTPS (при деплое) | ☐ | |

---

## 4. Фронтенд (UI)

| № | Проверка | Статус | Примечание |
|---|----------|--------|------------|
| 4.1 | Поле ввода пароля имеет тип `password` (символы скрыты) | ☐ | |
| 4.2 | Пароль не отображается на экране после проверки | ☐ | |
| 4.3 | Пароль не сохраняется в localStorage / sessionStorage | ☐ | |
| 4.4 | Пароль не виден в исходном коде страницы | ☐ | |

---

## 5. Общие проверки

| № | Проверка | Статус | Примечание |
|---|----------|--------|------------|
| 5.1 | Нет возможности отправить пароль GET-запросом | ☐ | |
| 5.2 | Нет хардкода паролей в коде | ☐ | |
| 5.3 | База утечек не содержит исходных паролей | ☐ | |
